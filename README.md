# CodePunk v2.0 Realtime Leaderboard

A comprehensive, realtime leaderboard and administration dashboard built for hackathons and coding competitions. This project provides a massive, public-facing presentation layer combined with a protected organizer control panel.

## Architecture & Tech Stack

- **Frontend Framework**: React 18, Vite
- **Styling**: Tailwind CSS v4
- **Animations**: Framer Motion
- **Database**: Firebase Firestore (NoSQL, Realtime Subscriptions)
- **Deployment**: Configured for Vercel / Netlify

## Key Features

1. **Real-time Synchronization** 
   - All score mutations broadcast to clients instantly via Firestore WebSockets.
2. **Multi-Track Segregation** 
   - Architecture supports decoupled tracks (e.g., Software vs Hardware) running simultaneously on the same database.
   - Dynamic round generation per track.
3. **Dense Rank Calculation**
   - Ranks are computed client-side. Ties are handled using a 'dense' ranking system (e.g., `#1, #2, #2, #3`).
4. **Administrative Controls**
   - Bulk CSV hydration mapping natively to dynamic columns.
   - Global event controls (Leaderboard Freeze state, Celebration Confetti trigger).
   - Dynamic round configurations (Add/Rename/Delete round columns on the fly).

## Database Architecture

The application relies on two primary Firestore collections:

### 1. `teams`
A record of all participating entities and their scores.

```typescript
{
  id: string; // Autogenerated document ID
  name: string;
  track: "software" | "hardware";
  scores: Record<string, number>; // Dynamic map: { [Round Name]: Score }
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### 2. `settings/globals`
A single document controlling the environment variables and stage directions for the live event.

```typescript
{
  rounds_software: string[]; // e.g. ["Ideation", "Technical", "UI/UX"]
  rounds_hardware: string[]; // e.g. ["Prototype", "Final"]
  isFrozen: boolean; // Overlays the public board with a pending screen
  celebrationAt: Timestamp; // Triggers synchronized confetti
  updatedAt: Timestamp;
}
```

## Security & Access Control

Mutations via the `/admin` portal are highly restricted. Admin permissions are structured as follows:

1. Enable **Authentication** in Firebase (Email/Password).
2. Create your organizer account.
3. Navigate to **Firestore Database** and create a collection called `admins`. 
4. Add a new document using your unique `uid` as the document ID. (The contents of the document can be empty).
5. Ensure `firestore.rules` is deployed so that only authenticated user IDs matching an `admins/uid` document are permitted write access.

## Setup & Local Development

1. **Clone and Install**
```bash
git clone <repository_url>
cd Leaderboard
npm install
```

2. **Environment Configuration**
Create a `.env.local` file at the root. Copy your Firebase config object variables into the following structure:
```bash
VITE_FIREBASE_API_KEY="your-api-key"
VITE_FIREBASE_AUTH_DOMAIN="your-auth-domain"
VITE_FIREBASE_PROJECT_ID="your-project-id"
VITE_FIREBASE_STORAGE_BUCKET="your-storage-bucket"
VITE_FIREBASE_MESSAGING_SENDER_ID="your-sender-id"
VITE_FIREBASE_APP_ID="your-app-id"
```

3. **Start the Development Server**
```bash
npm run dev
```

## Usage Guide: CSV Imports

The Admin dashboard allows for robust CSV ingestion to rapidly populate teams. 

- **Required Column Header**: `name` (or `Team Name`)
- **Track Separation**: Include a `track` column (`software` or `hardware`). If omitted, it defaults to software.
- **Score Mapping**: If you include headers that strictly match your currently active rounds (e.g. `Technical Interview`), the import engine will automatically map those values to the team's score.
